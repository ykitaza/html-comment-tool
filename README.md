# html-comment

任意の **ファイルをラップ**して、ブラウザでレビューしながらコメントを付け、
そのコメントを **AI への修正指示プロンプト**としてコピーできるローカルツールです。

[difit](https://github.com/yoshiko-pg/difit) のように `npx` 一発でローカルサーバが
立ち上がり、ブラウザが開きます。**元のファイルは一切変更しません**（読み込むだけ）。

## 対応フォーマットと 2 つのビュー

すべてのファイルに **ソースビュー**（生テキスト＋行番号）があります。一部の
ファイルはさらに **プレビュービュー**（見た目表示）を持ち、ツールバーのトグルで
Obsidian のように切り替えられます。

| 対象 | プレビュー（見た目） | ソース（生テキスト） |
|---|---|---|
| `.html` / `.htm` | iframe でレンダリング（要素クリック / テキスト選択） | HTML の生テキスト＋行番号 |
| `.md` / `.markdown` | HTML に変換して表示（要素クリック / テキスト選択） | Markdown の生テキスト＋行番号 |
| `.drawio` / `.dio` | drawio ビューアで図として表示（閲覧のみ／要ネット接続） | XML の生テキスト＋行番号 |
| `.json` `.yaml` `.xml` `.svg` `.txt` `.csv` ほか | （なし→ソースのみ） | 生テキスト＋行番号 |

- **位置情報**: プレビューでのコメントは CSS セレクタ（HTML）または
  **元 Markdown の行番号**（Markdown）、ソースでのコメントは `L42` / `L42-L48`。
- **コメントは両ビュー共通**: プレビューで付けた指摘もソースで付けた指摘も、
  同じパネルにまとまり、1 つのプロンプトに混在できます。
- **データパス**: JSON / YAML はソースで **データパス**（例: `services.web.ports`）も
  自動推定してプロンプトに添えます。AI が「どのキーの話か」を正確に把握できます。

> **drawio** はプレビューで drawio 公式ビューア（viewer.diagrams.net）を使って
> 図として表示します（閲覧のみ・ネット接続が必要）。図形への精密なコメントは
> ソース（XML）ビューで行えます。SVG は XML テキストとしてレビューします。

## なぜ作ったか

AI にファイルを生成してもらう機会は増えたものの、

- レビューは **レンダリング結果 / 整形された見た目**を見て行いたい
- でも指摘は **生のソース（要素・行）**に対して行う必要がある

このギャップが面倒でした。このツールは、見ている箇所を
自動的に CSS セレクタ／行番号などの位置情報に変換し、コメントとセットで
AI 向けプロンプトに整形します。

## インストール不要で実行（npx）

GitHub から直接実行できます（npm publish 不要）:

```bash
npx github:ykitaza/html-comment-tool ./design.html
```

npm に公開済みなら:

```bash
npx html-comment-tool ./design.html
```

ローカルにクローンして使う場合:

```bash
git clone https://github.com/ykitaza/html-comment-tool.git
cd html-comment-tool
node bin/cli.js ./path/to/file
```

## 使い方

```bash
html-comment ./design.html      # HTML（プレビュー⇄ソース）
html-comment ./spec.md          # Markdown（プレビュー⇄ソース）
html-comment ./config.yaml      # YAML（ソースのみ）
html-comment ./diagram.drawio   # drawio/XML（ソースのみ）
```

ブラウザが開いたら:

1. 必要なら **👁 プレビュー / `<>` ソース** を切り替える（対応ファイルのみ）
2. 左側で対象を選ぶ:
   - **プレビュー**: 要素をクリック、または「✎ テキスト選択」で文章をドラッグ
   - **ソース**: 行をクリック、または複数行をドラッグして範囲選択
3. その場でコメントを記入（⌘/Ctrl+Enter で確定）
4. 右パネルにコメントが溜まる（プレビューはピン、ソースは行番号ハイライト）
5. コメントを AI に渡す:
   - 各コメントの **📋（個別コピー）** で 1 件だけコピー
   - フッターの **「📋 AIプロンプトをコピー」** で全件まとめてコピー
6. コピーされた内容を **そのまま Claude Code などに貼り付け** → 修正してもらう

ダブルクリックでコメント編集、🗑 で削除できます。

### 主な機能（difit にならった点）

- **個別コピー / 一括コピー** … difit の "Copy Prompt" / "Copy All Prompt" 相当
- **永続化** … コメントはブラウザの `localStorage` に**対象ファイルの絶対パス単位**で保存。
  リロードしても、同じファイルを再度開いても復元されます。
- **フルパス出力** … プロンプトには対象ファイルの**絶対パス**が入るので、
  どのリポジトリ・どの場所のファイルか AI が迷いません。
- **ブラウザを閉じるとサーバも終了**（difit と同じ。`--keep-alive` で常駐可）。
- **ポート既定 4900**（使用中なら自動で次の空きポートへ）。

### オプション

```
-p, --port <n>   ポート指定（既定: 4900、使用中なら次の空きポートへ）
    --no-open    ブラウザを自動で開かない
    --clean      このファイルの保存済みコメントを消してから起動
    --keep-alive ブラウザを閉じてもサーバを維持
-h, --help       ヘルプ
```

## 出力されるプロンプトの例

**render モード（HTML）:**

````markdown
以下は `/abs/path/design.html` のレビューコメントです。コメントに従ってHTMLを修正してください。

## コメント
- 対象ファイル: `/abs/path/design.html`
- 対象セレクタ: `div.card:nth-of-type(3) > button.cta`
- 該当HTML:
```html
<button class="cta">設定を開く</button>
```
- 指摘 / 修正指示:
  ボタンの色をもっと目立つ色にしてください
````

**source モード（YAML、データパス付き）:**

````markdown
## コメント
- 対象ファイル: `/abs/path/config.yaml`
- 対象行: `L6-L8`
- データパス: `services.web.ports`
- 該当箇所:
```yaml
    ports:
      - "80:80"
      - "443:443"
```
- 指摘 / 修正指示:
  ポート公開を80のみに絞ってください
````

## 仕組み

- **render モード**: CLI が対象 HTML のディレクトリを web ルートとして配信し、
  ラッパーページと同一オリジン（localhost）の iframe で開きます。これにより
  iframe 内の DOM に自由にアクセスでき、要素・テキスト選択が可能です。
  セレクタは `id` 優先、なければ安定クラス + `:nth-of-type()` で一意に解決できる
  パスを生成します。
- **source モード**: 生テキストを `/__source` で受け取り、行番号付きで描画。
  行クリック / 行ドラッグで `L<n>` / `L<a>-L<b>` を確定します。
  JSON / YAML は構造を簡易解析して**データパス**を推定します（完全なパーサ
  ではなく実用的なヒューリスティック）。
- フロントは `core.js`（共通エンジン）＋ `render.js` / `source.js`（モード別
  アダプタ）＋ `boot.js`（モード判定）に分離。共通の永続化・コピー・プロンプト
  生成は core が担います。

## 制約

- render 対象 HTML が iframe 埋め込みを拒否する（`X-Frame-Options` 等）場合は
  ローカルファイルでは通常問題になりませんが、外部 URL を対象にする用途は想定していません。
- drawio / SVG は **XML テキストとしてのレビュー**であり、図としては描画しません。
- データパス推定は簡易実装です。アンカー・複数ドキュメント・複雑な YAML など
  一部ケースでは付かない／ずれることがあります（行番号と該当箇所は常に正確です）。
- コメントは保存キーに**絶対パス**を使います。ファイルを別の場所へ移動・改名すると
  別物として扱われ、コメントは復元されません（`--clean` でいつでもリセット可）。

## 依存

なし（Node.js 18+ の標準モジュールのみ）。
