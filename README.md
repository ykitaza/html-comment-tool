# html-comment

任意の **HTML ファイルをラップ**して、ブラウザでレンダリング結果を見ながらコメントを付け、
そのコメントを **AI への修正指示プロンプト**としてコピーできるローカルツールです。

[difit](https://github.com/...) のように `npx` 一発でローカルサーバが立ち上がり、ブラウザが開きます。
**元の HTML ファイルは一切変更しません**（iframe で読み込むだけ）。

## なぜ作ったか

AI に HTML を生成してもらう機会は増えたものの、

- レビューは **レンダリング結果**を見て行いたい
- でも指摘は **生の HTML（要素）**に対して行う必要がある

このギャップが面倒でした。このツールは、レンダリング結果上でクリック／テキスト選択した箇所を
自動的に CSS セレクタや HTML 断片に変換し、コメントとセットで AI 向けプロンプトに整形します。

## 使い方

```bash
npx html-comment ./design.html
# または
node bin/cli.js ./design.html
```

ブラウザが開いたら:

1. **要素を選択**モード（既定）でページ上の要素をクリック → コメントを記入
2. または **テキスト選択**モードで文章をドラッグ選択 → コメントを記入
3. 右パネルにコメントが溜まる（ピンがページ上に表示される）
4. コメントを AI に渡す:
   - 各コメントの **📋（個別コピー）** で 1 件だけコピー
   - フッターの **「📋 AIプロンプトをコピー」** で全件まとめてコピー
5. コピーされた内容を **そのまま Claude Code などに貼り付け** → 修正してもらう

ダブルクリックでコメント編集、🗑 で削除できます。

### 主な機能（difit にならった点）

- **個別コピー / 一括コピー** … difit の "Copy Prompt" / "Copy All Prompt" 相当
- **永続化** … コメントはブラウザの `localStorage` に**対象ファイルの絶対パス単位**で保存。
  リロードしても、同じファイルを再度開いても復元されます。
- **フルパス出力** … プロンプトには対象ファイルの**絶対パス**が入るので、
  どのリポジトリ・どの場所のファイルか AI が迷いません。
- **ブラウザを閉じるとサーバも終了**（difit と同じ。`--keep-alive` で常駐可）。
- **ポート既定 4900**（使用中なら自動で次の空きポートへ）。

### オプション

```
-p, --port <n>   ポート指定（既定: 4900、使用中なら次の空きポートへ）
    --no-open    ブラウザを自動で開かない
    --clean      このファイルの保存済みコメントを消してから起動
    --keep-alive ブラウザを閉じてもサーバを維持
-h, --help       ヘルプ
```

## 出力されるプロンプトの例

````markdown
以下は `design.html` のレビューコメントです。各コメントに従ってHTMLを修正してください。

## コメント 1
- 対象セレクタ: `div.card:nth-of-type(3) > button.cta`
- 該当HTML:
```html
<button class="cta">設定を開く</button>
```
- 指摘 / 修正指示:
  ボタンの色をもっと目立つ色にしてください
...
````

## 仕組み

- CLI が **対象 HTML のあるディレクトリを web ルート**として配信するため、
  対象が参照する相対パスの CSS/JS/画像もそのまま読み込まれます。
- ラッパーページと対象 HTML は **同一オリジン（localhost）** で配信されるので、
  iframe 内の DOM に自由にアクセスでき、要素選択・テキスト選択が可能です。
- セレクタは `id` を優先し、なければ安定したクラス名 + `:nth-of-type()` で
  **一意に解決できるパス**を生成します。

## 制約

- 対象 HTML が iframe 埋め込みを拒否する（`X-Frame-Options` を自分で付ける等）場合は
  ローカルファイルでは通常問題になりませんが、外部 URL を対象にする用途は想定していません。
- コメントは保存キーに**絶対パス**を使います。ファイルを別の場所へ移動・改名すると
  別物として扱われ、コメントは復元されません（`--clean` でいつでもリセット可）。

## 依存

なし（Node.js 18+ の標準モジュールのみ）。
